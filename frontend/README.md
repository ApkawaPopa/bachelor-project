# АМИНА - Frontend

Frontend часть защищенного мессенджера "АМИНА", реализованная на Vue.js 3 с использованием современных веб-технологий и
криптографии.

## Архитектура и технологии

- **Фреймворк:** Vue.js 3.5.25 (Composition API)
- **Сборщик:** Vite 7.2.4
- **Стилизация:** CSS Grid/Flexbox
- **Real-time общение:** SockJS + STOMP.js
- **Криптография:** Web Crypto API
- **HTTP клиент:** Fetch API

## Ключевые особенности

### End-to-End Шифрование

- Генерация RSA-2048 ключевых пар на клиенте
- Шифрование приватного ключа с использованием PBKDF2 и AES-GCM
- Симметричное шифрование сообщений AES-256-GCM
- Уникальный симметричный ключ для каждого чата

### Безопасная аутентификация

- Хранение JWT токена в localStorage
- Защищенное хранение ключевых материалов
- Автоматическое восстановление сессии

### Real-time общение

- WebSocket соединение через SockJS
- STOMP протокол для обмена сообщениями
- Автоматические подтверждения прочтения

### Пользовательский интерфейс

- Минималистичный дизайн с темной темой
- Адаптивная верстка с использованием CSS Grid
- Интуитивная навигация между чатами

## Быстрый старт

### Предварительные требования

- Node.js версии ^20.19.0 или ≥22.12.0
- Запущенный бэкенд сервер (см. README бэкенда)
- Современный браузер с поддержкой Web Crypto API

### Установка и запуск

1. **Установка зависимостей:**

```bash
cd frontend
npm install
```

2. **Настройка адреса бэкенда:**
   В файле `src/App.vue` измените переменную `address`:

```javascript
let address = "localhost:8080"  // или ваш адрес бэкенда
```

3. **Генерация SSL сертификатов (для разработки):**

```bash
# Генерация самоподписанного сертификата
openssl req -x509 -newkey rsa:2048 -keyout localhost.key \
  -out localhost.crt -days 365 -nodes -subj "/CN=localhost"
openssl pkcs12 -export -out localhost.pfx \
  -inkey localhost.key -in localhost.crt \
  -passout pass:changeit
```

4. **Запуск в режиме разработки:**

```bash
npm run dev
```

Приложение будет доступно по адресу: `https://localhost:3125`

5. **Сборка для production:**

```bash
npm run build
```

6. **Предпросмотр production сборки:**

```bash
npm run preview
```

## Криптографическая система

### Регистрация пользователя

1. Генерация RSA ключевой пары (2048 бит)
2. Экспорт публичного ключа в JWK формате
3. Шифрование приватного ключа:

- Используется пароль пользователя
- PBKDF2 для получения ключа шифрования
- AES-GCM для шифрования с случайным salt и IV
- Формат: версия(1 байт) + salt(16 байт) + IV(12 байт) + зашифрованный ключ

### Вход в систему

1. Дешифрование приватного ключа с использованием пароля
2. Импорт приватного ключа для использования в криптографических операциях
3. Получение зашифрованных симметричных ключей чатов
4. Дешифрование симметричных ключей с помощью приватного RSA ключа

### Шифрование сообщений

1. Каждое сообщение шифруется симметричным ключом чата
2. Формат зашифрованного сообщения:
   ```
   версия(1 байт) + IV(12 байт) + зашифрованное содержимое
   ```
3. Результат кодируется в base64 для передачи

### Создание чата

1. Генерация нового симметричного ключа AES-256-GCM
2. Получение публичных ключей всех участников
3. Шифрование симметричного ключа публичным ключом каждого участника
4. Отправка зашифрованных ключей на сервер

## WebSocket соединение

### Подключение

1. Получение WebSocket токена от бэкенда
2. Установление соединения через SockJS
3. Аутентификация с использованием JWT токена

### Подписки

- `/user/queue/error` - получение ошибок
- `/user/queue/chat` - получение уведомлений о новых чатах
- `/topic/chat/{id}` - получение сообщений из конкретного чата
- `/topic/chat/{id}/received` - получение подтверждений прочтения

### Отправка сообщений

- `/app/chat/{id}/message/post` - отправка нового сообщения
- `/app/chat/{chatId}/message/{messageId}/receive` - подтверждение прочтения

## Хранение данных

### LocalStorage

Приложение хранит следующие данные в браузере:

- `jwtToken` - JWT токен для аутентификации
- `username` - имя текущего пользователя
- `password` - пароль пользователя (только для сессии)
- `public` - публичный ключ в формате JWK
- `private` - зашифрованный приватный ключ в base64

### Безопасность хранения

- Приватный ключ хранится в зашифрованном виде
- Пароль хранится только во время сессии
- JWT токен имеет ограниченное время жизни

## Безопасность

### Рекомендации

1. **Никогда не храните пароли в открытом виде**
2. Используйте HTTPS в production
3. Регулярно обновляйте зависимости
4. Проводите аудит безопасности криптографических операций
5. Ограничивайте время жизни JWT токенов

### Ограничения

- Приватный ключ зависит от пароля пользователя
- Потеря пароля означает потерю доступа к зашифрованным данным
- Web Crypto API требует HTTPS в production

## Устранение неполадок

### Распространенные проблемы

1. **Ошибка подключения к бэкенду:**

- Проверьте адрес бэкенда в App.vue
- Убедитесь, что бэкенд запущен и доступен
- Проверьте настройки CORS на бэкенде

2. **Ошибки SSL:**

- Убедитесь, что сертификаты сгенерированы правильно
- Проверьте пароль сертификата в vite.config.js
- Для разработки можно временно отключить HTTPS

3. **Проблемы с WebSocket:**

- Проверьте, что получен действительный WebSocket токен
- Убедитесь, что бэкенд поддерживает WebSocket
- Проверьте настройки брандмауэра

4. **Ошибки криптографии:**

- Убедитесь, что браузер поддерживает Web Crypto API
- Проверьте форматы ключей и данных
- Проверьте последовательность криптографических операций

### Отладка

Включите логирование в консоли для отладки:

- Установите `console.log` в ключевых функциях
- Проверяйте Network вкладку в DevTools
- Используйте Vue DevTools для отладки состояния
